Visual Studio Live Share

实时协作开发

欢迎使用 Visual Studio Live Share!

只需简单几步即可让团队开始进行协作编辑和调试:

为什么要登录？

协作时，有必要让其他人能确定你是谁，这样他们才能放心地 共享对其代码和开发环境的访问权限。登录后，你将获得读写权限，从而能够与主持人共同编辑代码。

为便于入门， 你可使用现有 GitHub 或 Microsoft 帐户进行登录。

为便于入门， 你可使用现有 GitHub 或 Microsoft 帐户进行登录。

共享

单击右上角的 "Live Share" 按钮进行项目共享，再将唯一会话 URL 复制到剪贴板。

(使用 GitHub 或 Microsoft 帐户)首次共享时，你将需要登录，这样其他人才能在协作时确定你的身份。

在 Windows 上，系统可能会要求你允许 Live Share 打开防火墙端口，以实现对等连接。

编辑

你现在进行协作会话，你可移动光标、编辑和突出显示文本。你还能看到其他参与者在执行相同操作。

通过单击“已加入”下拉菜单或右上角的参与者图标，可以访问所有 Live Share 功能。

你可在此处查看会话参与者以及已共享的服务器和终端。

  • 移动到代码中的其他文件。可在项目中导航。

  • 关注参与者并提醒其关注你正在查看的代码片段。

  • 通过单击右上角的姓名跟随会话中的任何参与者。

调试

使用许多协作调试功能与你的团队实时完成问题/疑问/任务。

  •  在位于调试会话中时共同调试、设置断点，还可实时协作。

  • ☁️ 可查看共享服务器，方式是单击“已加入”下拉菜单来查看主持人在本地运行的 Web 应用和数据库等

  •  可显示共享终端，方式是单击“已加入”下拉菜单来查看或在主机的终端运行命令。

邀请

使用电子邮件、Skype 和 Slack 等将会话 URL 发送到你与之协作的开发人员(最多 5 名!)。要详细了解安全共享，请查看安全信息。

${this.joinUri}

想要评估 VS Live Share 但未与任何人协作? 没问题! 实际上，你可与其他 VS 实例中的自己进行共享。请单击此处进行尝试。

协作

搞定! 受邀“来宾”加入后，你将立即收到通知，并将在当前打开的文件中看到其光标。所有来宾都将开始跟随你，这意味着在你滚动查看或在文件间导航时，这些来宾也将跟踪这些操作。这样，你可以轻松地向他们介绍你要开始协作处理的问题/难点/任务。

此外，为充分利用 Live Share，请尝试以下几项内容:

  • 移动光标、编辑和突出显示文本，还可观看来宾执行同样的操作
  • 跟踪来宾，让其滚动和/或导航到其他文件
  • 共享调试会话，方式是按 F5，再实时同步协作
  • 共享服务器，让来宾能够查看在本地运行的 Web 应用和数据库等
  • 共享终端并协同运行命令

如果遇到任何问题，请参阅我们的疑难解答文档了解潜在的解决方法。此外，如有任何疑问和/或反馈，请随时使用“帮助”菜单中的“发送反馈”选项联系相关团队。
