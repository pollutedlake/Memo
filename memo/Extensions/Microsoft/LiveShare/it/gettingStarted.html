Visual Studio Live Share

Sviluppo con collaborazione in tempo reale

Benvenuti in Visual Studio Live Share

Bastano pochi semplici passaggi per consentire al team di iniziare a usare le funzionalità di modifica e debug in collaborazione:

Perché eseguire l'accesso?

Quando si collabora, è importante comunicare la propria identità in modo che gli altri utenti sappiano di poter condividere l'accesso al codice e all'ambiente di sviluppo in condizioni di sicurezza. Dopo l'accesso si disporrà dei diritti di lettura e scrittura e sarà possibile modificare il codice in collaborazione con l'organizzatore.

Per iniziare facilmente, è possibile accedere usando un account GitHub o Microsoft esistente.

Per iniziare facilmente, è possibile accedere usando un account GitHub o Microsoft esistente.

Condividi

Fare clic sul pulsante Live Share nell'angolo in alto a destra, per condividere il progetto e copiare un URL di sessione univoco negli Appunti.

La prima volta che si condivide codice, con un account GitHub o Microsoft, verrà chiesto di eseguire l'accesso in modo che gli altri possano riconoscere l'utente durante la collaborazione.

In Windows è possibile che venga chiesto di consentire a Live Share di aprire una porta firewall per consentire connessioni peer-to-peer.

Modifica

Dopo l'accesso alla sessione di collaborazione è possibile muovere il cursore, modificare ed evidenziare il testo, nonché vedere altri partecipanti eseguire le stesse operazioni.

Per accedere a tutte le funzionalità Live Share, fare clic sul menu a discesa "Accesso eseguito" oppure sulle icone dei partecipanti in alto a destra.

Da qui è possibile vedere i partecipanti alla sessione, nonché i server e i terminali che sono stati condivisi.

  • Passa ad altri file nel codice. È possibile spostarsi all'interno del progetto.

  • Richiamare l'attenzione di un partecipante su una parte del codice che si sta osservando.

  • Per seguire i partecipanti di una sessione, fare clic sul relativo nome in alto a destra.

Debug

Collaborare con il team a un problema, una domanda o un'attività in tempo reale usando le numerose funzionalità di debug collaborativo.

  •  Eseguire il debug insieme e, durante una sessione di debug, impostare i punti di interruzione e eseguire il passaggio insieme in tempo reale.

  • ☁️ Per visualizzare i server condivisi, fare clic sul menu a discesa "Accesso eseguito" in modo da visualizzare l'app Web, i database e altri elementi dell'organizzatore eseguiti in locale.

  •  Per visualizzare i terminali condivisi, fare clic sul menu a discesa "Accesso eseguito" in modo da visualizzare o eseguire comandi nel terminale dell'organizzatore.

Collabora

Tutto qui! Dopo che il partecipante invitato avrà accettato, si riceverà una notifica e il suo cursore apparirà nel file attualmente aperto. Si verrà quindi seguiti da tutti i partecipanti, di conseguenza quando si scorre o ci si sposta tra i file, queste azioni verranno visualizzate anche dai partecipanti. In questo modo potranno orientarsi più facilmente in merito al problema, alla domanda e all'attività su cui si sta per iniziare a collaborare.

Inoltre, per sfruttare al massimo Live Share, ecco alcuni suggerimenti da provare:

  • Spostare il cursore, modificare ed evidenziare testo e guardare i partecipanti eseguire le stesse operazioni
  • Seguire un partecipante e chiedergli di scorrere e/o passare ad altri file
  • Condividere una sessione di debug, premendo F5 e quindi procedendo insieme in tempo reale
  • Condividere un server, in modo che gli ospiti possano visualizzare un'applicazione Web o un database in esecuzione in locale.
  • Condividere un terminale ed eseguire comandi in collaborazione

In caso di problemi, consultare la documentazione sulla risoluzione dei problemi per le possibili soluzioni. Per domande e/o commenti, contattare il team usando le opzioni Commenti e suggerimenti del menu Guida.
