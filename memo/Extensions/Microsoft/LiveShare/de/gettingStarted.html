Visual Studio Live Share

Gemeinschaftliche Entwicklung in Echtzeit

Willkommen bei Visual Studio Live Share!

Mit einigen wenigen Schritten ermöglichen Sie Ihrem Team den Einstieg in das gemeinsame Bearbeiten und Debuggen:

Zweck der Anmeldung

Bei der Zusammenarbeit ist es wichtig, dass andere Personen Sie identifizieren können, um den Zugriff auf ihren Code und ihre Entwicklungsumgebung sicher freigeben zu können. Nach der Anmeldung verfügen Sie über Lese-/Schreibrechte, sodass Sie den Code zusammen mit dem Gastgeber bearbeiten können.

Zum einfachen Einstieg können Sie sich mit einem vorhandenen GitHub- oder Microsoft-Konto anmelden.

Zweck der Anmeldung

Bei der Zusammenarbeit ist es wichtig, dass andere Personen Sie identifizieren können, um den Zugriff auf ihren Code und ihre Entwicklungsumgebung sicher freigeben zu können. Nach der Anmeldung können Sie Ihren Arbeitsbereich freigeben und mit anderen zusammenarbeiten.

Zum einfachen Einstieg können Sie sich mit einem vorhandenen GitHub- oder Microsoft-Konto anmelden.

Freigeben

Klicken Sie in der oberen rechten Ecke auf die Schaltfläche Live Share. Dadurch wird Ihr Projekt freigeben und eine eindeutige Sitzungs-URL in die Zwischenablage kopiert.

Sie werden bei der ersten Freigabe aufgefordert, sich über ein GitHub- oder Microsoft-Konto anzumelden, damit andere Benutzer Sie bei der Zusammenarbeit identifizieren können.

Unter Windows werden Sie möglicherweise aufgefordert, Live Share das Öffnen eines Firewallports zu gestatten, um Peer-zu-Peer-Verbindungen zu ermöglichen.

Bearbeiten

Sie befinden sich nun in einer Zusammenarbeitssitzung und können Ihren Cursor bewegen sowie Text bearbeiten und hervorheben. Sie können beobachten, wie andere Teilnehmer die gleichen Tätigkeiten ausführen.

Sie können auf alle Live Share-Funktionen zugreifen, indem Sie auf das Dropdownmenü "Beigetreten" oder oben rechts auf das Teilnehmersymbol klicken.

Hier werden Teilnehmer der Sitzung sowie freigegebene Server und Terminals angezeigt.

  • Fahren Sie mit anderen Dateien im Code fort. Sie haben die Möglichkeit, im Projekt zu navigieren.

  • Senden Sie eine Fokusanforderung an einen Teilnehmer, und lenken Sie seine Aufmerksamkeit auf einen Teil des Codes, den Sie gerade betrachten.

  • Folgen Sie einem beliebigen Teilnehmer in einer Sitzung, indem Sie oben rechts auf den entsprechenden Namen klicken.

Debuggen

Bearbeiten Sie mithilfe der zahlreichen Debugfunktionen für die Zusammenarbeit gemeinsam mit Ihrem Team bestimmte Probleme, Fragen oder Tasks in Echtzeit.

  •  Führen Sie in einer Debugsitzung das Debugging gemeinsam aus, legen Sie Haltepunkte fest, und führen Sie Einzelschritte in Echtzeit aus.

  • ☁️ Zeigen Sie freigegebene Server an, indem Sie auf das Dropdownmenü "Beigetreten" klicken, um die lokal ausgeführten Web-Apps, Datenbanken usw. des Hosts anzuzeigen.

  •  Zeigen Sie freigegebene Terminals an, indem Sie auf das Dropdownmenü "Beigetreten" klicken, um im Hostterminal Befehle anzuzeigen oder auszuführen.

Einladen

Senden Sie die Sitzungs-URL per E-Mail, Skype, Slack etc. an die Entwickler, mit denen Sie zusammenarbeiten möchten (bis zu 5!). Zeigen Sie Sicherheitsinformationen an, um weitere Informationen zur sicheren Freigabe zu erhalten.

Zusammenarbeiten

Das war's! Sobald ein eingeladener "Gast" der Sitzung beitritt, erhalten Sie eine Benachrichtigung und sehen seinen Cursor in der aktuell geöffneten Datei. Alle Gäste folgen Ihnen anfangs. Das bedeutet, dass die Gäste Ihre Aktionen verfolgen und Sie beim Scrollen oder Navigieren zwischen Dateien beobachten können. So können sie sich ganz einfach über das Issue, die Frage oder die Aufgabe orientieren, an der Sie zusammenarbeiten möchten.

Um Live Share optimal zu nutzen, können Sie außerdem folgende Aktionen ausprobieren:

  • Verschieben Sie den Cursor, bearbeiten und markieren Sie Text, und beobachten Sie, wie Gäste dieselben Aktionen durchführen.
  • Folgen Sie einem Gast, und lassen sie ihn scrollen oder zu anderen Dateien navigieren.
  • Geben Sie eine Debugsitzung frei, indem Sie F5 drücken und dann die Einzelschritte in Echtzeit ausführen.
  • Geben Sie einen Server frei, sodass Gäste eine lokal ausgeführte Web-App, Datenbank usw. anzeigen können.
  • Geben Sie ein Terminal frei, und führen Sie Befehle gemeinsam aus.

Wenn Probleme auftreten, finden Sie mögliche Problemumgehungen in unserer Dokumentation zur Problembehandlung. Wenn Sie Fragen und/oder Feedback haben, können Sie sich darüber hinaus über die Optionen Feedback senden im Menü Hilfe an das Team wenden.
