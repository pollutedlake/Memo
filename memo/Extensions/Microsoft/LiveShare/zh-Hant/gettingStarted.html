Visual Studio Live Share

即時共同開發

歡迎使用 Visual Studio Live Share!

只要幾個簡單步驟，就能讓小組開始共同編輯與偵錯:

為什麼要登入?

在共同作業時，重要的是其他人可以識別您的身分，以便放心 共用其程式碼與開發環境的存取權。登入後，就會有讀寫權限，因此您可以與主持人共同編輯程式碼。

為方便您開始使用， 您可使用現有的 GitHub 或 Microsoft 帳戶登入。

為什麼要登入?

在共同作業時，重要的是其他人可以識別您的身分，以便放心 共用其程式碼與開發環境的存取權。登入後，就可以共用您的工作區並與其他人共同作業。

為方便您開始使用， 您可使用現有的 GitHub 或 Microsoft 帳戶登入。

共用

按一下右上角的 [Live Share] 按鈕，這會共用您的專案，並將唯一的工作階段 URL 複製到剪貼簿。

系統會在您初次共用時要求登入 (使用 GitHub 或 Microsoft 帳戶)，讓其他人在共同作業時知道您的身分。

在 Windows 上，系統可能會要求您允許 Live Share 開啟防火牆連接埠，以使用對等連線。

編輯

您現已加入共同作業工作階段，您可在其中移動游標、進行編輯以及將文字醒目提示。您可以看到其他參與者執行上述動作。

按一下 [已加入] 下拉式功能表或右上方的參與者圖示，即可存取所有 Live Share 功能。

您可以從這裡看到工作階段中的參與者，以及已共用的伺服器與終端機。

  • 移至程式碼中的其他檔案。您可以在專案中巡覽。

  • 聚焦參與者，並請他注意您目前所查看程式碼的一部份。

  • 在右上方按一下工作階段中任何參與者的名稱，即可追蹤參與者。

偵錯

使用為數眾多的共同偵錯功能，來和您的小組一同解決問題/工作。

  •  即時在偵錯工作階段中共同偵錯，以及共同設定中斷點及執行步驟。

  • ☁️ 檢視共用伺服器，方法是按一下 [已加入] 下拉式功能表，來檢視主持人在本機執行的 Web 應用程式、資料庫等。

  •  顯示共用終端機，方法是按一下 [已加入] 下拉式功能表，來檢視或執行主持人終端機內的命令。

邀請

使用電子郵件、Skype、Slack 等，將工作階段 URL 傳送給您想要共同作業的開發人員 (最多 5 人!)。檢視安全性資訊以深入了解安全的共用。

要評估 VS Live Share，但沒有其他人可以共同作業嗎? 不用擔心! 您其實可以從 VS 的另一個執行個體與自己共用。按一下這裡即可試用。

共同作業

這樣就行了! 只要受邀的「來賓」加入，您就會收到通知，並會看到他們的游標出現在您目前開啟的檔案中。所有來賓都會開始跟隨您，這表示當您捲動或在檔案之間瀏覽時，他們也會追蹤這些動作。如此即可輕鬆將他們導向您即將開始共同作業的問題/工作。

此外，若要充分運用 Live Share，可嘗試進行下列作業:

  • 移動您的游標、編輯及醒目提示文字，然後看來賓執行相同動作
  • 追蹤來賓，並將它們捲動及 (或) 瀏覽至其他檔案
  • 共用偵錯工作階段，方法是按 F5，然後即時一起逐步執行
  • 共用伺服器，以便來賓可檢視在本機執行的 Web 應用程式、資料庫等
  • 共用終端機及共同執行命令
